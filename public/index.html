Prototypes

  • Prototypes allow us to implement inheritance in JS
  • They are not Classes!
  • A prototype is an object from which other objects inherit properties

JS Objects

Prototypes

  • A property/method added to a prototype will be available to all it's instances
  • A property/method added to one of it's instances will not be available the prototype object or the other instances
  • You can overwrite a property/method in an instance

JS Objects

The prototype chain

Prototype inheritance works by going up the prototype chain if the called method/property is not found in the current object.

When the method is found, it won't look further.

JS Objects

Object.create( )

With Object.create( ) we use an object as the prototype for other objects.

This new object will inherit its properties from the prototype object.

New properties/methods added to the prototype object will be available in all created objects. (from that prototype)

JS Objects

Object.create( )

Creating the prototype Object:

  var person = {
    name: "no name",
    gender: "no gender",
    init: function(name, gender){
      this.name = name;
      this.gender = gender;
    },
    greet: function(){
      console.log("Hello my name is " + this.name);
    }
  };

JS Objects

Object.create( )

Creating an instance:

  var person1 = Object.create(person);
  person1.init("Paulo", "m");
  person1.greet(); //"Hello my name is Paulo"

JS Objects

Constructor

A constructor is a function used for initializing new objects.

You use the new keyword to call the constructor.

It has an associated prototype object that can be accessed: funcName.prototype

JS Objects

Constructor

  var Person = function(name, gender){
    this.name = name;
    this.gender = gender;
    this.greet = function(){
      console.log("Hello my name is " + this.name);
    };
  };

JS Objects

Constructor

  var person1 = new Person("Paulo", "m");
  person1.greet(); //"Hello my name is Paulo"
  Person.prototype; // *prototype*

JS Objects

Inheritance with Constructor:

  var Employee = function(name, gender, jobTitle){
   Person.call(this, name, gender);
   this.jobTitle = jobTitle;
  };
  Employee.prototype =  Object.create(Person.prototype);

JS Objects

Factory

Easy to understand.

Avoids "this" problems.

Flexible.

JS Objects

Factory

  var PersonFactory = function(name, gender){
    var tmp = {};
    tmp.name = name;
    tmp.gender = gender;
    tmp.greet = function(){
      console.log("Hello my name is " + tmp.name);
    };
    return tmp;
  };

JS Objects

Factory

  var person1 = PersonFactory("Paulo","m");
  person1.greet(); //"Hello my name is Paulo"

JS Objects

Composition

With Inheritance you decide you types based on what they ARE.

With Composition you decide you types based on what they DO.

JS Objects

Factory Composition

  var PersonFactory = function(state, name, gender){
    state.name = name;
    state.gender = gender;
    state.greet = function(){
      console.log("Hello my name is " + state.name);
    };
    return state;
  };

JS Objects

Factory Composition

  var RobotFactory = function(state, model, battery){
    state.model = model;
    state.battery = battery;
    state.move = function(){ this.battery -= 20; };
    state.isLowOnBattery = function(){ 
      return this.battery <= 30;
    };
    return state;
  };

JS Objects

Factory Composition

  var batman = PersonFactory( {}, "Bruce Wayne", "m");
  
  var spaceRobot = RobotFactory( {}, "r2d2", 60);

  var robocop = PersonFactory( {}, "Alex J. Murphy", "m");
  robocop = RobotFactory( robocop,
    "OCP Crime Prevention Unit 001", 100);
